  <files>.claude/agents/first-principles.md</files>
  <action>
Write the First Principles framework agent definition. This agent:

1. Breaks down any problem/approach to its fundamental truths
2. Questions every assumption — "What do we KNOW vs what do we ASSUME?"
3. Rebuilds from ground truth upward

The agent prompt must follow the Agent Communication Protocol from CLAUDE.md:
- Role: "You are a First Principles thinking agent"
- Receives: CONTEXT (current state), SUBJECT (what to analyze), INPUT_FILES (paths to read)
- Applies 3 core questions:
  1. What are the fundamental truths here? (not assumptions, not conventions)
  2. What assumptions are being made that could be wrong?
  3. If we rebuilt from scratch knowing only the fundamentals, what would we do differently?
- Output format: For each question: PRINCIPLE → EVIDENCE → CONCLUSION
- Ends with: VERDICT (PASS/FAIL) + specific findings + recommendations
- Must cite source files for every claim (evidence protocol)

Make the agent generic — it must work for ANY domain (regulatory, software, data, etc.), not just ProSure.

Key lesson from Mission 1086: This framework discovered 32 retired EBA metrics that the main pipeline missed — by asking "what IS a DataPoint at the fundamental level?"

<task type="auto">
  <name>Task 3: Create 5-Whys framework agent</name>
  <files>.claude/agents/five-whys.md</files>
  <action>
Write the 5-Whys framework agent definition. This agent:

1. Drills to root cause by asking "Why?" iteratively (up to 5 levels)
2. Distinguishes symptoms from causes
3. Identifies the deepest actionable root cause

The agent prompt must follow the Agent Communication Protocol:
- Role: "You are a 5-Whys root cause analysis agent"
- Receives: CONTEXT, SUBJECT (the problem or gap to analyze), INPUT_FILES
- Applies iterative drilling:
  1. State the problem clearly
  2. Why 1: [first-level cause] — EVIDENCE: [citation]
  3. Why 2: [deeper cause] — EVIDENCE: [citation]
  4. Why 3: [deeper still] — EVIDENCE: [citation]
  5. Why 4: [approaching root] — EVIDENCE: [citation]
  6. Why 5: [root cause] — EVIDENCE: [citation]
  (Stop early if root cause found before level 5)
- Output format: Problem → Why chain with evidence → ROOT CAUSE → actionable fix
- Ends with: VERDICT (PASS/FAIL) + root cause + recommended action
- Every "Why" level must cite evidence — no speculation

Key lesson from Mission 1086: 5-Whys found 4 misclassified BoE 6000-series items where a numeric heuristic was wrong — drilling through "why are these unclassified?" revealed the flawed assumption.
  </action>

- Role: "You are an Occam's Razor simplicity agent"
- Receives: CONTEXT, SUBJECT, INPUT_FILES

- Ends with: VERDICT (PASS/FAIL) + simplification recommendations
- Must cite evidence for "observed facts" — no assumed facts
