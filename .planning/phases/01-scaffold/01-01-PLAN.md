<task type="auto">
  <name>Task 2: Create plugin CLAUDE.md with conventions and architecture</name>
  <files>CLAUDE.md</files>
  <action>
Create CLAUDE.md documenting the plugin's architecture and conventions:

1. **Plugin Overview** — One paragraph: what this plugin does, extracted from Mission 1086 methodology
2. **Architecture** — The 4-layer model:
   - Framework Agents (8 built-in, user-extensible): independent thinking lenses
   - T1 Validator: pre-execution approach validation using N frameworks
   - Cycle Auditor: per-cycle quality audit dispatching framework agents
   - Evidence Protocol: zero uncited inference enforcement
3. **Agent Communication Protocol** — Every agent prompt MUST include:
   - Objective (single unambiguous sentence starting with a verb)
   - Input files (exact paths to read)
   - Output file (exact path to write)
   - Schema (expected structure)
   - Verification (how to confirm success)
   - Constraints (what NOT to do)
4. **Orchestrator Accountability** — Before presenting ANY result:
   - Verify agent output exists and is valid
   - Verify counts match expectations
   - Verify no null/empty required fields
   - Cross-check agent claims against actual data
5. **Framework Agent Verdicts** — Standard output format:
   - T1: VALIDATED / VALIDATED_WITH_ADJUSTMENTS / REJECTED
   - Per-Cycle: PASS / FAIL with specifics per framework
6. **Custom Frameworks** — How users add their own framework agents (drop .md in agents/)

Keep concise. This is a working reference, not documentation.
  </action>
  <verify>cat CLAUDE.md shows all 6 sections; wc -l CLAUDE.md is under 120 lines</verify>
  <done>CLAUDE.md exists with all 6 sections, under 120 lines, accurately reflects Mission 1086 methodology</done>
</task>
