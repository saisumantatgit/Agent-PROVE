Create the evidence-audit skill with proper YAML frontmatter and workflow.

Frontmatter:
- name: evidence-audit
- description: Audit any output for evidence protocol compliance — zero uncited inference allowed
- Arguments: TARGET (required, file path or inline text to audit), --strict (optional, fail on any warning), --source-dir (optional, directory to verify citations against)

Skill workflow:
1. Read the TARGET (file or inline text)
2. For each claim/statement in the target:
   a. Check if it cites a source (file path, URL, section reference, data point)
   b. If source cited, verify the source exists (if --source-dir provided)
   c. If source cited but doesn't exist, flag as BROKEN_CITATION
   d. If no source cited, flag as UNCITED_INFERENCE
3. For each number/statistic in the target:
   a. Check if it has a derivation (how was this number computed?)
   b. If no derivation, flag as UNVERIFIED_NUMBER
4. Compile audit report:
   - Total claims analyzed
   - Claims with valid citations
   - UNCITED_INFERENCE violations (with line numbers)
   - BROKEN_CITATION warnings (with line numbers)
   - UNVERIFIED_NUMBER warnings (with line numbers)
   - Overall verdict: COMPLIANT / NON_COMPLIANT
5. If --strict: any warning = NON_COMPLIANT
6. Present results with specific line numbers and suggested fixes

The evidence standard (from Mission 1086 methodology):
- Every classification must cite source_file + xpath/location
- Every legal authority must cite instrument_name + article + URL + in_force_status
- "Genuinely unassigned" items must have exhaustive search evidence (which files were searched, what was NOT found)
- The generic version: every factual claim needs [source], every number needs [derivation], every "not found" needs [search evidence]

Include usage examples:
- `/audit data/output.json` — check output for evidence compliance
- `/audit @docs/report.md --strict` — strict mode, any warning = fail
- `/audit @data/classification.json --source-dir data/taxonomy/` — verify citations exist
